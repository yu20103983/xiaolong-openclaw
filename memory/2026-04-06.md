## Optimizations Completed (all committed & pushed to GitHub main+master)

### Speed Optimizations
1. **Gateway Bridge event-driven connect** — `bridge_ready` event replaces 3s hardcoded sleep in agent_client.py + gateway-bridge.js
2. **Bluetooth vs local adaptive delays** — `BT_SWITCH_DELAY`, `BT_SILENCE_PREFIX`, `PRE_PLAY_DELAY`, `POST_PLAY_DELAY` — local devices get near-zero delays
3. **TTS cache hint fix** — removed trailing period from "好的，我来处理一下。" → "好的，我来处理一下" to fix cache miss
4. **ASR worker thread timeout** — 0.5s → 0.05s in asr_engine.py
5. **Parallel initialization** — ASR, TTS, Agent init in parallel threads (saves 2-3s startup)
6. **Removed steer 0.5s sleep** — no delay after sending system prompt
7. **Smart input timeout** — short commands (几点/播放/天气 etc.) use 1.5s timeout; others keep 4s. Config: `INPUT_QUICK_TIMEOUT = 1.5`
8. **Collector queue timeout** — sentence_queue.get reduced from 0.3s to 0.1s for faster first-sentence TTS
9. **Interrupt abort wait** — reduced from 1s to 0.3s after user says "终止"

### Stability/Robustness
10. **TTS three-tier fallback** — Edge TTS → Matcha-TTS (local sherpa-onnx) → Windows SAPI. Auto-switches after 3 consecutive edge failures. Config: `TTS_ENGINE`, `TTS_LOCAL_MODEL`
11. **Agent auto-restart steer recovery** — Bridge crash restart now re-sends system prompt
12. **Windows QuickEdit protection** — `_disable_quickedit()` prevents console click freeze
13. **Dynamic TTS results saved to disk cache** — synthesized audio persisted for reuse across restarts

### UX Improvements
14. **Sleep command bug fix (CRITICAL)** — `_is_sleep_command()` had empty body; its logic was dead code after `_extract_after_long()` return. Fixed.
15. **Extended wake word variants** — `_XIAO_CHARS`: 16 chars (added 销削校效啸歇). `_LONG_CHARS`: 15 chars (added 东). Strict subset `_LONG_STRICT` (龙隆笼聋拢龍) for single-char matching to prevent false triggers.
16. **Extended sleep expressions** — 12+ natural phrases: 不说了/不聊了/拜拜/先这样/就这样/没事了/去休息/下次再说/打扰了 etc.
17. **Processing feedback** — When user sends command while Agent is working, plays "收到，等我处理完当前任务" and queues the command.
18. **README updated** — reflects all new features, correct config examples, TTS fallback FAQ.

### Rejected Optimizations (with reasons)
- VAD silence 0.5s→0.3s: risk of cutting speech mid-sentence
- handle_command 0.1s sleep removal: race condition risk on slow devices
- TTS cached.copy() removal: marginal gain (~μs), risk if playback ever mutates array
- Dynamic TTS disk cache on every synth: IO overhead, low hit rate for unique sentences (later added anyway as part of TTS refactor)
